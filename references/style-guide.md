# 标准 Word 样式规范

本文件是 `standard-word-doc` 的机器可读权威规范。脚本在没有
`assets/standard-word-template.docx` 时必须按本规范创建 `.docx`；审计和修复
脚本也必须按本文件的检查项执行。

## 核心原则

- 格式一致性优先于内容丰富度。
- 有模板时继承模板；无模板时使用本文件默认样式。
- 不依赖 Word/WPS/LibreOffice/Pandoc 才能生成基础 `.docx`。
- 不在正文中手写章节编号，不用加粗正文模拟标题。
- 不手动插入 Unicode bullet 字符，必须使用 Word 列表样式。
- 不使用 Symbol、Wingdings、Wingdings 2、Wingdings 3 字体。

## API 接口详细设计结构

生成 API 详细设计文档时，每个接口优先按以下结构输出：

| 小节 | 内容 |
| --- | --- |
| 接口基本信息 | 接口名称、方法、路径、模块、鉴权、Content-Type、幂等性 |
| 请求参数 | 参数名、位置、类型、必填、说明、示例、约束 |
| 请求体字段 | 字段名、类型、必填、默认值、说明、校验规则 |
| 响应字段 | 字段名、类型、说明、示例 |
| 状态码和错误码 | HTTP 状态码、业务错误码、含义、处理建议 |
| 示例报文 | 请求 JSON、响应 JSON，使用代码块而不是普通正文 |
| 业务规则 | 权限、限流、重复提交、数据一致性、异常处理 |
| 测试要点 | 正常路径、异常参数、权限失败、重复提交、边界值 |

## 格式检查项

### 错误级别

| ID | 检查项 | 修复策略 |
| --- | --- | --- |
| E001 | Unicode bullet 字符作为段落开头 | 删除符号，应用 `List Bullet` 样式 |
| E002 | 标题使用正文样式加粗和大字号模拟 | 按字号映射到 Heading 1-4 |
| E003 | 使用 Symbol/Wingdings 字体 | 替换为 Arial/微软雅黑 |
| E004 | 段落中存在硬换行 `<w:br>` 或 `\v` | 拆分为独立段落 |

### 警告级别

| ID | 检查项 | 修复策略 |
| --- | --- | --- |
| W001 | 正文字号出现两种以上不同值 | 统一为 11pt |
| W002 | 表格宽度为百分比 | 转为 DXA 固定宽度 |
| W003 | 连续两个以上空段落 | 仅保留一个空段落 |
| W004 | 手动编号写在正文段落中 | 仅报告，需确认是否改为标题 |
| W005 | 标题层级跳跃 | 仅报告，不自动修复 |

### 建议级别

| ID | 检查项 | 修复策略 |
| --- | --- | --- |
| S001 | 正文存在全角空格 | 仅报告，建议替换为半角 |
| S002 | 中英文之间没有空格 | 仅报告，避免误改专有名词 |
| S003 | 文档缺少页码字段 | 仅报告，建议添加或刷新页脚 |

## 审计报告要求

审计报告必须包含：

- 文件路径
- 问题总数
- 错误、警告、建议分级统计
- 每条问题的位置，优先使用段落索引、标题文字或表格索引
- 每条问题的规则 ID、说明和建议处理方式

## 修复输出要求

- 不覆盖原始文件。
- 修复后的 `.docx` 文件名追加 `_repaired`。
- 同时生成 `_repair_summary.md`，说明自动修复项和仍需人工确认项。
